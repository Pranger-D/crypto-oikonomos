Phase 1. 기초 공사 (안전 장치 및 환경 설정) ★지금 당장 할 것
코드를 짜기 전에 '어디에 무엇을 넣을지' 정리하고, '올리지 말아야 할 것'을 막는 단계입니다.

1-1. 폴더 구조 정리: 웹(Next.js)과 봇(Python)이 섞이지 않게 폴더를 나눕니다.

1-2. .gitignore 설정: 가상환경(venv)이나 API 키(.env)가 Github에 올라가 해킹당하거나 꼬이는 것을 방지합니다. (가장 중요)

1-3. Python 가상환경(venv) 생성: 이 프로젝트만을 위한 독립된 파이썬 방을 만듭니다.

Phase 2. 자동화 엔진 개발 (Python Core)
블로그 디자인보다 **'시스템'**이 먼저입니다. 글이 자동으로 올라가는지부터 확인해야 합니다.

2-1. 콘텐츠 생성기 (Writer): Python으로 "제목, 날짜, 본문"이 담긴 가짜 뉴스(Mock Data)를 Markdown 파일로 만드는 기능 구현.

2-2. 깃허브 업로더 (Courier): 내 컴퓨터의 Python이 Github API를 통해 원격 저장소로 파일을 쏘아 올리는 기능 구현.

2-3. 통합 테스트: Python 스크립트 실행 한 번으로 -> Github 업로드 -> Vercel 배포까지 되는지 확인.

2-4. 뉴스 소스 연동 (Real Data): 가짜 뉴스 대신 실제 경제 뉴스 API나 크롤링 연동.

Phase 3. 블로그 최적화 (Optimization)
시스템이 돌아가면 그때부터 집을 꾸밉니다.

3-1. 메타데이터 및 SEO: siteMetadata.js 수정, 조회수 잘 나오는 키워드 세팅.

3-2. UI/UX 개선: 한글 폰트 적용, 가독성 개선, 메뉴 정리.

Phase 4. 수익화 (Monetization)
